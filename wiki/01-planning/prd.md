## TL;DR
- **무엇**: AI가 노트를 직접 조작하는 데스크톱 노트 앱. AI는 **NVIDIA NIM 무료 API**(사용자 본인 키)로 구동. 1차는 **텍스트 작성 도구**.
- **누구**: 설치·구독 부담 없이 AI 노트를 바로 쓰고 싶은 지식 노동자·개발자·학생.
- **성공**: "NIM 키 입력 → 질문 → AI가 노트에 직접 작성"의 왕복이 안정적으로 동작하고, 노트는 로컬에 저장되며, 설치파일로 배포된다.

## 1. 배경 / 문제
AI 노트 도구는 대개 **월 구독**을 요구하거나, 로컬 실행 방식은 **설치·GPU·모델 다운로드** 장벽이 크다.
"가볍게, 돈 안 들이고, 바로" AI가 노트를 써 주는 선택지가 없다.
→ **무료 API 키 하나만 입력하면** 강력한 모델이 노트를 직접 써 주는 데스크톱 앱.

## 2. 타겟 유저
- **1차 타겟**: 데스크톱에서 노트를 많이 쓰는 개인. 설치·구독은 피하고 싶지만, **NVIDIA 무료 API 키 발급** 정도는 할 수 있는 얼리어답터.
- **동기**: "구독도 무거운 설치도 없이, 내 키로 무료로 AI 노트를 쓰고 싶다."

## 3. 목표 & 성공 기준 (MVP)
- [기능] NIM API 키 입력 후, 채팅으로 요청하면 AI가 **노트 본문에 직접 텍스트를 작성/수정**한다.
- [안정] 노트 생성·편집·삭제·검색 + **로컬 저장**이 데이터 손실 없이 동작한다.
- [배포] Windows 설치파일(.msi/.exe)로 실행 가능한 빌드가 나온다.
- [정성] 사용자가 "설치·구독 없이 바로 AI 노트가 된다"를 체감한다.

## 4. 범위 — 1차 MVP (In scope)
1. **노트 관리**: 생성 / 편집 / 삭제 / 검색, **로컬 저장**(파일 또는 SQLDelight+SQLite).
2. **편집기**: 제목 + 본문(플레인/마크다운), 자동 저장.
3. **NIM 연동**: API 키 입력 · **안전 저장(OS 자격증명 저장소/암호화)** · 연결 검증 · 모델 선택 · 상태 표시(상단 바).
4. **AI 도우미 패널**: 모델과의 대화형 인터페이스.
5. **텍스트 작성 도구(tool calling)**: AI가 노트에 `삽입 / 이어쓰기 / 선택 부분 수정 / 요약해 넣기`.
6. **ai-mark**: AI가 쓴/고친 부분을 시각적으로 구분(보라 흔적) — DESIGN.md 시그니처.
7. **라이트/다크 테마**.

## 5. 핵심 사용자 시나리오
1. 회의 메모를 붙여넣고 "핵심만 정리해줘" → AI가 요약을 노트에 넣고, 그 부분에 ai-mark 표시.
2. 글을 쓰다 "이 문단 더 매끄럽게 고쳐줘" → 선택 부분을 AI가 수정.
3. 빈 노트에서 "이 주제로 개요 잡아줘" → AI가 뼈대를 작성.
4. (최초 1회) 설정에서 NVIDIA NIM API 키를 입력 → 이후 자동 사용.

## 6. 비범위 (Out of scope — 2차 이후)
- **그림/다이어그램 캔버스**(excalidraw식) → 2차 핵심.
- 로컬 실행(Ollama) 옵션, 클라우드 동기화·계정, 모바일 앱, 실시간 협업, 플러그인.
- 고급 편집(첨부, 표, 임베드)은 MVP 이후.

## 8. 기술 전제
- Kotlin Multiplatform + Compose Multiplatform (Desktop/JVM).
- 로컬 저장: SQLDelight + SQLite (초기엔 파일 기반 가능).
- AI: **NVIDIA NIM**(OpenAI 호환 API, `integrate.api.nvidia.com/v1`) + **사용자 API 키**. 도구 스키마는 앱이 정의·실행.

## 9. 가정 & 리스크 (※ 확인 필요)
- **가정 A — 플랫폼**: 1차는 **Windows 우선**(사용자 환경), 이후 mac/Linux. → 맞나요?
- **가정 B — 키 발급**: 사용자가 NVIDIA 무료 API 키를 직접 발급·입력. 앱은 발급 안내 + 입력 UI 제공.
- **가정 C — 수익화**: 무료 MVP, 이후 2차(그림) 등을 Pro로. → 8단계에서 확정.
- **리스크 1 — 무료 티어 한도**: NIM 무료는 요청/크레딧 한도가 있을 수 있음 → 초과 안내·재시도.
- **리스크 2 — 네트워크 의존**: 오프라인 시 AI 기능 정지(노트 편집은 가능).
- **리스크 3 — API 키 보안**: 키를 로그·커밋·화면에 노출 금지, 안전 저장 필수.
- **리스크 4 — 모델별 tool 지원 차이**: 도구 호출 지원 모델을 권장/기본값으로.

## 10. 대략 마일스톤
- **M1** 노트 CRUD + 로컬 저장 + 편집기.
- **M2** NIM 키 연결/검증 + AI 채팅(도구 없이 일반 대화).
- **M3** 텍스트 작성 도구(tool calling) + ai-mark.
- **M4** 패키징/배포(.msi) + 온보딩(키 발급 안내).
