## TL;DR
- **무엇**: 로컬 LLM(Ollama)이 노트를 직접 조작하는 데스크톱 노트 앱. 1차는 **텍스트 작성 도구**.
- **누구**: 데이터를 클라우드에 보내기 싫은, AI 도움은 받고 싶은 지식 노동자·개발자·학생.
- **성공**: "Ollama 연결 → 질문 → AI가 노트에 직접 작성"의 왕복이 로컬에서 안정적으로 동작하고, 설치파일로 배포된다.

## 1. 배경 / 문제
기존 AI 노트 앱은 대부분 **클라우드 전송**을 전제한다. 민감한 메모·업무 노트를 다루는 사용자는
프라이버시 때문에 AI 도움을 포기하거나, 복사-붙여넣기로 우회한다.
→ **완전 로컬에서 AI가 노트를 직접 써 주는** 도구가 없다.

## 2. 타겟 유저
- **1차 타겟**: 프라이버시를 중시하는 개인 지식 노동자 / 개발자 / 대학원생. 데스크톱에서 노트를 많이 쓰고,
  Ollama를 설치할 의향/능력이 있는 얼리어답터.
- **동기**: "내 기기 안에서만 도는 AI 노트가 필요하다."

## 3. 목표 & 성공 기준 (MVP)
- [기능] Ollama 연결 후, 채팅으로 요청하면 AI가 **노트 본문에 직접 텍스트를 작성/수정**한다.
- [안정] 노트 생성·편집·삭제·검색 + 로컬 저장이 데이터 손실 없이 동작한다.
- [배포] Windows 설치파일(.msi/.exe)로 실행 가능한 빌드가 나온다.
- [정성] 사용자가 "클라우드 없이 AI 노트가 된다"를 체감한다.

## 4. 범위 — 1차 MVP (In scope)
1. **노트 관리**: 생성 / 편집 / 삭제 / 검색, 로컬 저장(파일 또는 SQLDelight+SQLite).
2. **편집기**: 제목 + 본문(플레인/마크다운), 자동 저장.
3. **Ollama 연동**: 모델 선택, 연결 상태 표시(상단 바), 미연결 시 안내.
4. **AI 도우미 패널**: 로컬 모델과의 대화형 인터페이스.
5. **텍스트 작성 도구(tool calling)**: AI가 노트에 `삽입 / 이어쓰기 / 선택 부분 수정 / 요약해 넣기`.
6. **ai-mark**: AI가 쓴/고친 부분을 시각적으로 구분(보라 흔적) — DESIGN.md 시그니처.
7. **라이트/다크 테마**.

## 5. 핵심 사용자 시나리오
1. 회의 메모를 붙여넣고 "핵심만 정리해줘" → AI가 요약을 노트에 넣고, 그 부분에 ai-mark 표시.
2. 글을 쓰다 "이 문단 더 매끄럽게 고쳐줘" → 선택 부분을 AI가 수정.
3. 빈 노트에서 "이 주제로 개요 잡아줘" → AI가 뼈대를 작성.
4. (상시) 상단 바에서 로컬 모델 연결 상태를 확인, 모든 처리는 기기 내에서.

## 6. 비범위 (Out of scope — 2차 이후)
- **그림/다이어그램 캔버스**(excalidraw식) → 2차 핵심.
- 클라우드 동기화·계정·로그인, 모바일 앱, 실시간 협업, 플러그인/확장.
- 고급 편집(첨부, 표, 임베드)은 MVP 이후.

## 7. 화면
- 4영역 레이아웃(상단 상태바 · 좌측 노트목록 · 중앙 편집기 · 우측 AI 패널).
  상세: [화면 구성](../02-design/screens.md) · 목업 [ui-mockup.html](../02-design/ui-mockup.html).

## 8. 기술 전제
- Kotlin Multiplatform + Compose Multiplatform (Desktop/JVM).
- 로컬 저장: SQLDelight + SQLite (초기엔 파일 기반 가능).
- AI: Ollama `/api/chat` 의 tool calling. 도구 스키마는 앱이 정의·실행.

## 9. 가정 & 리스크 (※ 확인 필요)
- **가정 A — 플랫폼**: 1차는 **Windows 우선**(사용자 환경), 이후 mac/Linux. → 맞나요?
- **가정 B — Ollama 설치**: 사용자가 Ollama를 직접 설치·실행. 앱은 연결만. → 온보딩 안내 필요.
- **가정 C — 수익화**: 무료 MVP, 이후 2차(그림) 등을 Pro로. → 8단계에서 확정.
- **리스크 1 — 작은 로컬 모델의 tool calling 신뢰도**: 도구 호출 실패/오작동 가능 → 권장 모델 명시 + 실패 시 폴백(일반 답변).
- **리스크 2 — 성능/메모리**: 로컬 추론 부하 → 최소 사양 정의.

## 10. 대략 마일스톤
- **M1** 노트 CRUD + 로컬 저장 + 편집기.
- **M2** Ollama 연결 + AI 채팅(도구 없이 일반 대화).
- **M3** 텍스트 작성 도구(tool calling) + ai-mark.
- **M4** 패키징/배포(.msi) + 온보딩.
